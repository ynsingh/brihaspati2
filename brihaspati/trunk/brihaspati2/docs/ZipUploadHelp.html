Multiple User Registration With a Single Zip File

"Add Multiple Users" registration page is available to the institute admin, provides an interface to create the course of primary Instructor and student.
This module allows an Institute Admin to register multiple user as Primary instructor and as students along with their courses into the system using single zip file.
To effectively use of feature we should remember all the necessary conditions and instructions which are given below.

1. Guideline for creating registration file of instructor

  • File name must be "inst.txt"
  • The authorised user should write down Full name of Instructor, Email Id of instructor, Course Id and Course Name in the "inst.txt" file which are all separated by semi-colons.
  • Each line must contain four fields. Each field separated by semicolon(;).
  • Note that there is no semicolon after the CourseName.
  • Hence, the required format for instructor is :-
    Full name of instructor; Email Id; Course Id; Course Name
  • Example:- Tej B Gurung; tejb@iitk.ac.in; TEST101; Testing of Modules

2. Guideline for creating registration file of student
  • File name must be written with courseid*(see Note).
  • Example:- TEST101.txt
  • The authorised user should write down the email, password, firstname, familyname, programcode, rollno and course_id**(see Note).
  • All field separated by semi-colons in the student's registeration files.
  • Note that there is no semicolon(;) after the course_id.
  • Hence, theere are two required format for student is:-
    1. *  email; password; firstname; familyname; programcode; rollno; course_id
    • 2. **  email; password; firstname; familyname; programcode; rollno

  • Note-    * If file name is not a course id then course id must be written in the file after roll no field.
    • Example:- tejb@iitk.ac.in; ;tej;bahadur; ;y1001;AE101

  • Note-    ** Course Id is optional If file name is same as course id.
  • Example:- tejb@iitk.ac.in; ;tej;bahadur; ;y1001
3. Important Instructions
  • It is necessary that each file should be kept in a folder before zip the file.
  • Each field in the entry should consist of alphanumeric characters. Email field is execption to this. '@' and '_' are allowed for use in this field.
  • Email and Roll No are mandatory(*) fields and Course Id**(see Note) is optional.
  • Without these fields, the new entry into the system will not be registered.
  • Similarly, an entry containing an illegal character will also not be registered into the system. Example:- # % & * { } \ : < > ? / +
  • Each entry must be in one line and must contain the six fields (seventh field is optional**(see Note)) separated by six delimiters (;) in the Student's registeration file.
  • Ensure that there is no need of semicolon(;) after Course name field in instructor's registeration file and Course Id field in the student's registeration file.
