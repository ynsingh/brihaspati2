Table Asks for Information to be filled about Remote Institute who is interested to buy or sell course.
Following are the Table Details :
  1. I.P Address -- IP address of Remote Institute
  2. Institute Name -- Institute Name of Remote Institute
  3. Course ID -- Course Id of course of Instructor in Remote Institute
  4. Instructor Name -- Name of Instructor(Login Name) in Remote Institute
  5. Secret Key -- Secret Key to be agreed by both party
  6. Transaction -- Buy/Sell
  7. Expiry Date -- Expiry date upto which course can be accessed

How To Use Remote Course

    Suppose you (Instructor of Course) want to sell/buy your course content to instrutor of another Institute using Brihaspati LMS.
  1. Fill the Remote IP Address and click on the Get button to get Institute name.
  2. Select Institute Name and click on Get button to get Course list.
  3. Now Select that CourseId in which you(Course Instructor) want to sell/buy.
  4. Now Click the Get button to get Login Name.
  5. Fill the Secret Key field and press submit.Now you will be shown your entry (highlighted among other entries).If you feel that the entry is not correct ,click on update link.Correct the entries and press update.Now again you will be shown your entry.
  6. Next you have to check if the remote Institute has registered you in his brihaspati server or not .To verify this click on Check-Remote-Registration link.If you are registered and you are buyer than you click on Get - Course - content link .
  7. If you get message that course was send succesfully click on link Course Management than on Course Content.Now you will see that requested Remote Course Id contents are available for use.
  8. Your course are now in publish and unpublished area as in remote side.Once your subscription has expired you have to re-buy/update the Expiry date on both side else you will lose the links to remote courses.

Known limitations of Remote Courses

  1. Remote Courses works only on port 8080.
  2. Those courses which are obtained from others via permission or remote course (BUY) cannot be selled.
  3. Until you register IP ADDRESS and Course ID of Remote user with local server you cannot sell/buy any course.
  4. If the remote server is not running you cannot access remote course.
  5. If the port 12345 is not alive remote courses access can not be done.
